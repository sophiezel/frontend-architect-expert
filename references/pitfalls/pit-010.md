# pit-010: scroll-snap 在 iOS 上的滚动对齐异常

## 环境
- iOS Safari（12+ 支持 scroll-snap）
- 结合 `overflow: scroll` 或 `-webkit-overflow-scrolling: touch`

## 症状
`scroll-snap` 在 iOS Safari 上滚动停止位置不精确，或滑动惯性导致跳过多个 snap 点，出现"跳格子"效果。用户体验差，轮播图无法精确停在某一页。

## 根因
iOS Safari 的 scroll-snap 实现在惯性滚动（momentum scroll）阶段与 snap 点的交互存在问题：
1. `scroll-snap-type: x mandatory` + 快速滑动 → 惯性跳过中间 snap 点。
2. iOS 的橡皮筋效果（bounce）导致 snap 点计算偏移。
3. 如果 snap 容器内子元素尺寸动态变化，iOS 重新计算 snap 点不及时。
4. `-webkit-overflow-scrolling: touch` 与 `scroll-snap` 存在兼容性冲突。

## 修复

```css
/* ❌ 问题代码 */
.carousel {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  -webkit-overflow-scrolling: touch; /* 与 scroll-snap 冲突 */
}
.carousel-item {
  flex: 0 0 100%;
  scroll-snap-align: start;
}

/* ✅ 修复 */
.carousel {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  /* 移除 -webkit-overflow-scrolling: touch */
  overscroll-behavior-x: contain; /* 防止橡皮筋效果干扰 */
}
.carousel-item {
  flex: 0 0 100%;
  scroll-snap-align: start;
  scroll-snap-stop: always; /* iOS 15.4+ 防止跳过 snap 点 */
}
```

```diff
// ✅ 替代方案：JS 方案回退（兼容旧 iOS）
import { useRef, useCallback } from 'react';

function Carousel({ children }) {
  const containerRef = useRef(null);
  const scrollToIndex = useCallback((index) => {
    const container = containerRef.current;
    const item = container.children[index];
    container.scrollTo({
      left: item.offsetLeft,
      behavior: 'smooth'
    });
  }, []);
  // ...
}
```

## 原理
iOS Safari 在触摸滚动时分为两个阶段：触摸驱动阶段（用户手指在屏幕上）和惯性阶段（手指离开后的动量滚动）。scroll-snap 在两个阶段的衔接处寻找最近的 snap 点。问题在于：

1. 惯性阶段的速度计算基于 `-webkit-overflow-scrolling: touch` 的旧滚动模型，与 CSS scroll-snap 的新模型冲突。
2. 橡皮筋效果在 snap 边界处会产生额外的偏移量，干扰 snap 点计算。
3. `overscroll-behavior: contain` 可以阻止橡皮筋效果传递到 snap 容器。

## 架构视角 (Architecture Perspective)

### 架构层面根因
scroll-snap 异常的架构根因是**原生 CSS 特性被直接暴露而非通过抽象层使用**：
- **轮播/分页是交互模式而非 CSS 属性**：轮播图、分页滚动是一种交互模式（Interaction Pattern），有明确的产品语义（每页一个内容块、支持手势切换、指示器同步）。但代码中将这个交互模式直接用 CSS `scroll-snap` 实现，没有封装为 `<Carousel>` 组件——导致不同页面有不同实现，兼容性问题散落各处。
- **交互模式缺少降级策略**：当一个交互模式（轮播）有多种实现方案（CSS scroll-snap、JS scrollTo、第三方库、原生 ScrollView）时，架构应该定义"首选方案 → 降级方案 → 兜底方案"的策略链。`scroll-snap` 应是首选，但如果检测到 iOS 14 以下或不支持 `scroll-snap-stop`，应自动降级到 JS 方案。
- **平台兼容性矩阵未纳入交互组件选型**：选择使用原生 CSS scroll-snap 时，需要审阅其平台兼容性矩阵（iOS Safari 12-14 不稳定，15.4+ 改善），并决定是否值得为不稳定版本编写额外的 polyfill。这个决策应在组件选型阶段完成，而非在各组件中逐个修复。

### 系统级预防策略

**静态防护**：
```js

rules: {
  'property-disallowed-list': ['scroll-snap-type', 'scroll-snap-align'],
  // 强制通过 <Carousel> / <PageScroll> 组件使用

```

```css
.scroll-snap-x {
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  overscroll-behavior-x: contain;
}

.scroll-snap-item {
  scroll-snap-align: start;
}

@supports (scroll-snap-stop: always) {
  .scroll-snap-item {
    scroll-snap-stop: always;
  }
}
```

**架构防护**：
- **交互模式库**：建立 `@app/interaction-patterns` 包，封装所有交互模式（Carousel、PullToRefresh、SwipeableList、ParallaxScroll），每个模式内部包含：
  - 首选实现（原生 CSS/浏览器 API）
  - 降级实现（JS 方案）
  - 特性检测逻辑（`CSS.supports` 或运行时探测）
  - 统一的组件接口
- **平台兼容性决策矩阵**：参考 `decisions.md` 中的格式，为每个交互模式建立兼容性矩阵，明确各平台的降级路径。

**流程防护**：

### 跨层影响链

```
上游影响 (消费者):
  - 父页面的滚动体验被轮播组件破坏（嵌套滚动冲突）
  - 页面级别的 scroll 事件监听器收到意外的滚动事件
  - 页面过渡动画与轮播动画时序冲突

下游影响 (依赖):
  - 轮播指示器（dot indicator）基于错误的 scroll 位置计算
  - 懒加载逻辑（检测当前可见 page → 加载数据）在 snap 不对齐时触发错误
  - A/B 测试框架的页面曝光统计在 snap 不对齐时重复或遗漏计数

部署影响:
  - iOS Safari 版本升级（如 14→15.4）改变了 scroll-snap 行为 → 需要回归测试
  - 降级方案（JS 方案）的 bundle 体积增加 → 影响性能指标
  - 使用第三方轮播库替代原生实现 → 引入新的安全漏洞风险

监控影响:
  - RUM: 按平台/版本分组监控滚动停止位置偏差
  - 自定义事件: snap 对齐成功/失败率
  - E2E: iOS Safari 真机测试（无法用 Chrome DevTools 模拟）
  - Session Replay: 观察用户滑动轮播时的异常行为（来回滑动、跳过内容）
```

### 相关架构模式
- **组件组合模式 — Compound Components** (`patterns.md` 第二节) — 轮播图天然适合 Compound Components（`<Carousel>` + `<Carousel.Slide>` + `<Carousel.Indicator>`），隐式共享当前索引状态。
- **构建工具选型** (`decisions.md` 第四节) — Vite/Rolldown 的 tree-shaking 可确保降级方案（JS 方案）仅在需要时打包。
- **CSS 方案选型** (`decisions.md` 第三节) — 轮播组件的样式方案需要支持特性检测（`@supports`）和渐进增强，CSS-in-JS 方案（运行时）在此场景下不如静态 CSS 可靠。

## 预防

1. **不要混用 `-webkit-overflow-scrolling: touch` 和 `scroll-snap`**。
2. **添加 `overscroll-behavior: contain`** 消除橡皮筋效果干扰。
3. **在 iOS 15.4+ 使用 `scroll-snap-stop: always`**。
4. **选择成熟的第三方轮播库**（如 Swiper.js），它们内部处理了大量 iOS 兼容细节。
5. **如果必须兼容 iOS 12-14**，使用 JS 方案（scrollTo + behavior: 'smooth'）代替原生 scroll-snap。
